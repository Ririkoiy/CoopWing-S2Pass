# Co-opWinG 使用说明

🌐 Language: 简体中文

> 当前文档对应 Co-opWinG 预览版。
> 功能、界面和打包方式仍可能变化。

---

## 1. 开始之前

Co-opWinG 是一个实验性的联机网络辅助工具，主要用于帮助部分合作游戏、旧游戏或点对点联机场景改善连接问题。

当前版本主要用于：

* 局域网 / 远程联机排障
* UDP 桥接与中继回退测试
* Windows 预览版体验

它不是 VPN，也不是虚拟局域网工具，不会安装虚拟网卡或系统驱动。

---

## 2. 适用场景

Co-opWinG 可能适用于：

* 游戏需要通过 TCP/UDP 端口通信
* 直连失败，但双方都可以访问同一个中继服务器
* 一方处于校园网、CGNAT 或严格 NAT 环境
* 游戏本身支持通过指定地址或本地端口连接
* 你知道或可以测试出游戏需要使用的端口

Co-opWinG 不保证适用于：

* 强依赖官方服务器匹配的游戏
* 完全加密且不可配置连接目标的游戏
* 需要虚拟局域网广播发现的游戏
* 需要协议级解析或修改的复杂游戏
* 明确禁止第三方网络工具的游戏场景

---

## 3. 基本概念

### 房间

Co-opWinG 使用“房间”来建立双方连接。

通常一名玩家创建房间，另一名玩家输入房间码加入。

### 后端

Co-opWinG 的界面会连接本地后端。后端负责网络会话、适配器和中继相关逻辑。

正常情况下，用户不需要手动操作后端。

### 适配器

适配器负责把游戏的本地网络流量接入 Co-opWinG。

### 中继

如果直接 P2P 连接失败，Co-opWinG 可以尝试通过中继服务器转发流量。

中继性能取决于服务器位置、带宽、延迟和双方网络环境。

---

## 4. 典型使用流程

### 创建房间的一方

1. 启动 Co-opWinG。
2. 等待后端状态变为可用。
3. 进入房间页面。
4. 点击创建房间。
5. 将房间码发送给另一名玩家。
6. 配置游戏或适配器参数。
7. 启动游戏或本地桥接。
8. 观察连接状态和流量显示。

### 加入房间的一方

1. 启动 Co-opWinG。
2. 等待后端状态变为可用。
3. 进入房间页面。
4. 输入对方提供的房间码。
5. 点击加入房间。
6. 配置游戏或适配器参数。
7. 启动游戏或本地桥接。
8. 观察连接状态和流量显示。

---

## 5. 测试建议

首次测试时，建议不要直接上复杂游戏。

推荐测试顺序：

1. 两台电脑都能正常启动 Co-opWinG。
2. 双方能成功创建 / 加入房间。
3. 后端状态正常。
4. 适配器状态正常。
5. 流量计数有变化。
6. 再测试具体游戏。

## 6. 常见问题

### 后端没有启动怎么办？

可以尝试：

* 重新打开 Co-opWinG
* 检查是否被杀毒软件拦截
* 检查 Windows 防火墙提示
* 确认程序目录没有被移动或缺少文件
* 查看是否已有旧的后端进程残留或尝试使用Stop-CoopWing-Backend.bat

### 创建或加入房间失败怎么办？

可以检查：

* 双方是否使用同一个服务器配置
* 房间码是否正确
* 网络是否可以访问中继 / 信令服务器
* 防火墙是否阻止程序联网
* 后端状态是否正常

### 游戏没有连接成功怎么办？

可以检查：

* 游戏端口是否正确
* 游戏是否真的使用 UDP 或 TCP 本地端口
* 游戏是否允许手动连接地址
* 适配器配置是否与游戏端口一致
* 双方是否都启动了对应适配器
* 是否存在防火墙拦截

### 为什么有流量但游戏还是连不上？

可能原因包括：

* 游戏需要多个端口
* 游戏需要协议感知型适配器
* 游戏还有官方服务器校验
* 游戏使用了动态端口
* 游戏流量方向和配置不一致
* 游戏本身不支持这种连接方式

---

## 7. Windows 防火墙提示

第一次运行时，Windows 可能会弹出防火墙提示。

如果你要进行局域网或远程联机测试，通常需要允许 Co-opWinG 或相关后端程序访问网络。

如果不允许，可能会出现：

* 创建房间失败
* 加入房间失败
* 本地端口无法接收数据
* 适配器无流量
* 游戏无法连接

---

## 8. 反馈问题时请提供

提交问题时，请尽量提供：

* Windows 版本
* Co-opWinG 版本
* 使用的是创建房间还是加入房间
* 双方网络环境，例如家庭宽带、校园网、移动热点、CGNAT
* 测试的游戏或工具
* 配置的端口
* 是否看到流量变化
* 是否触发中继
* 复现步骤
* 日志或截图


## 9. 当前限制

当前预览版仍有这些限制：

* 主要面向 Windows
* 不是所有游戏都支持
* 可能需要手动配置端口
* 中继质量依赖服务器和网络环境
* UI 和文案仍在调整
* 部分功能仍偏测试用途

当前版本仅为预览测试，而非稳定版。
